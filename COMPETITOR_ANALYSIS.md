## 1. 直接竞品

### 1.1 Ultralytics Platform

| 维度 | 详情 |
|------|------|
| **定位** | YOLO 官方全流程云平台 |
| **发布时间** | 2026 年 3 月正式发布 |
| **许可** | 部分开源 + 商业云服务 |
| **部署** | 云优先（43 个全球区域），提供免费额度 |

**功能矩阵：**
- 标注：SAM 3 驱动的点击式标注
- 训练：22 种 GPU 选项（A100、H100、RTX 4090）
- 导出：17 种格式（ONNX、TensorRT、CoreML、TFLite 等）
- 部署：自动扩缩容的生产级 API 端点
- 监控：内置生产监控仪表板

**与我们对比：**

| 维度 | Ultralytics Platform | VisionWorkbench |
|------|---------------------|-----------------|
| 部署 | 云优先 | **本地优先** |
| 费用 | 免费额度 + 付费 GPU | **完全免费** |
| 数据隐私 | 数据上传到云 | **数据完全本地** |
| 离线使用 | 不支持 | **完全支持** |
| 目标用户 | 专业开发者 | **学生和创客** |
| 学习曲线 | 需要理解云平台概念 | **大一学生可理解** |
| AI 原生设计 | 无明确 AI Agent 接口 | **AI Agent 是头等公民** |
| 实验对比 | 基本支持 | **核心模块** |
| 模型血统追踪 | 基本支持 | **完整链路追踪** |

**竞争策略：** 差异化——强调本地、免费、隐私、学生友好。不正面竞争，而是服务 Ultralytics 不关注的群体。

### 1.2 Fovux

| 维度 | 详情 |
|------|------|
| **定位** | 本地优先 YOLO 工作台，MCP 协议驱动 |
| **发布时间** | v1.0.0（2026 年 6 月） |
| **许可** | 开源 |
| **部署** | 本地（Python MCP Server + VS Code 扩展） |

**功能矩阵：**
- 训练：`train_start`、`train_status`、`train_stop`
- 数据集：`dataset_inspect`、`dataset_stats`
- 评估：`eval_compare`、`eval_run`
- 导出：`export_onnx`、`export_tensorrt`、`export_tflite`
- 架构：MCP 协议 + VS Code Studio 前端

**与我们对比：**

| 维度 | Fovux | VisionWorkbench |
|------|-------|-----------------|
| 交互方式 | MCP 协议 / VS Code / CLI | **独立 GUI 应用（Web UI）** |
| 用户门槛 | 需要懂 MCP 和 VS Code | **打开浏览器即用** |
| 数据集管理 | 基本检查 | **完整注册、校验、版本、统计** |
| 模型注册 | 无 | **完整版本管理、血统追踪** |
| 实验管理 | 基本比较 | **完整实验记录、对比、笔记** |
| Demo 功能 | 无 | **图片/视频/摄像头一键演示** |
| AI Agent | MCP 原生（面向协议） | **面向用户的 AI 辅助** |
| 适用人群 | 熟悉 MCP 的开发者 | **所有学生和创客** |

**竞争策略：** 差异主要在交互模式——Fovux 是面向开发者的 MCP 工具，VisionWorkbench 是面向所有人的 GUI 工作台。两者可能在高级用户群中有重叠，但核心用户体验完全不同。

### 1.3 YoloEase

| 维度 | 详情 |
|------|------|
| **定位** | 桌面端 YOLO 训练 GUI 工具 |
| **发布时间** | v2.0.90（2026 年 5 月） |
| **许可** | 开源 |
| **部署** | 本地桌面应用 |

**功能矩阵：**
- 标注小批量 → 训练 → 自动标注 → 审核 → 重训练
- 自动环境管理（Python、PyTorch、CUDA）
- 导出 .pt 和 .onnx

**与我们对比：**

| 维度 | YoloEase | VisionWorkbench |
|------|----------|-----------------|
| 覆盖环节 | 标注 + 训练 | **数据集→训练→实验→模型→评估→演示** |
| 交互式标注 | 支持 | **非核心（集成外部工具）** |
| 数据集管理 | 无结构化管理 | **完整数据集中心** |
| 实验管理 | 无 | **完整实验中心** |
| 模型注册 | 无 | **完整模型注册表** |
| 使用场景 | 为游戏自动化训练模型 | **通用 CV 研发** |

**竞争策略：** YoloEase 面向特定场景（游戏自动化），VisionWorkbench 面向通用 CV 研发工作流，覆盖范围更广。

### 1.4 VLM-AutoYOLO

| 维度 | 详情 |
|------|------|
| **定位** | VLM 驱动的自动标注 + YOLO 训练平台 |
| **发布时间** | v1.5.11（2026 年 6 月） |
| **许可** | AGPL-3.0 |
| **部署** | 自托管（FastAPI + PostgreSQL + React） |

**功能矩阵：**
- 自动标注：VLM（NVIDIA LocateAnything-3B）+ SAM2/SAM3
- 训练：YOLOv8/v11/v26
- Web UI：React + TypeScript

**与我们对比：**

| 维度 | VLM-AutoYOLO | VisionWorkbench |
|------|-------------|-----------------|
| 核心卖点 | **自动标注（VLM+SAM）** | **完整工作流管理** |
| 技术栈 | FastAPI + PostgreSQL + React | Streamlit/Gradio + SQLite |
| 部署复杂度 | 需要 PostgreSQL | **仅 SQLite，零配置** |
| 实验管理 | 无 | **核心模块** |
| 模型注册 | 无结构化注册 | **完整注册表** |
| 学生友好度 | 有部署门槛 | **一键启动** |

**竞争策略：** VLM-AutoYOLO 在自动标注方面很强，但 VisionWorkbench 胜在全面性和易用性。

## 2. 间接竞品

### 3.3 SWOT 分析

**优势 (Strengths)：**
- 唯一本地全流程 CV 工作台
- 零配置启动体验
- 学生和创客定价（免费）
- AI 原生架构设计
- 数据完全本地，隐私安全

**劣势 (Weaknesses)：**
- 全新项目，无用户基础
- 单一开发者或小团队
- 依赖 Streamlit/Gradio 的 UI 限制
- 初期功能深度不如专业工具

**机会 (Opportunities)：**
- CV 教育市场快速增长
- 竞赛团队对组织化工具需求强烈
- OpenMV/OpenART 生态缺乏配套工具
- AI Agent 集成是未来趋势
- 本地优先趋势在隐私意识增强下持续增长

**威胁 (Threats)：**
- Ultralytics 可能增强本地功能
- Fovux 可能推出更友好的 GUI
- Roboflow 可能加强离线支持
- 开源社区可能出现类似项目

## 4. 差异化总结

VisionWorkbench 的核心差异化不在于单一功能的深度，而在于：

1. **统一性**：6 个模块无缝连接，形成完整工作流
2. **本地性**：完全本地运行，无需网络
3. **简单性**：学生可以理解和使用
4. **AI 原生**：为 AI 辅助设计的架构
5. **可追溯性**：从数据集到模型的完整血统追踪

**一句话定位**：CVAT 管标注，YOLO 管训练，Excel 管实验，文件夹管模型——VisionWorkbench 一个工具管全部。
