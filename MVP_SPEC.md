# MVP_SPEC.md — VisionWorkbench

## 概述

本文档定义 VisionWorkbench 从 v0.1 到 v1.0 的 MVP 规格，遵循**每个版本必须独立可用**的渐进交付原则。

---

## 总体架构原则

1. **单进程架构**：所有模块在一个 Python 进程中运行
2. **文件系统 + SQLite**：数据存储在本地文件系统和 SQLite 中
3. **Web UI**：通过 Streamlit 提供浏览器界面
4. **零配置**：`pip install visionworkbench && vw start` 即可运行
5. **AI 可读**：所有数据以人类可读和 AI 可读的格式存储

## v0.1 — 单模型图片推理

### 目标

### 功能需求

| ID | 需求 | 优先级 | 说明 |
|----|------|--------|------|
| F0.1.1 | 加载模型 | P0 | 支持加载 .pt 格式的 YOLO 模型文件 |
| F0.1.2 | 图片推理 | P0 | 对单张图片执行目标检测推理 |
| F0.1.3 | 显示结果 | P0 | 在界面上显示带标注框的推理结果 |
| F0.1.4 | 保存结果 | P0 | 将推理结果保存为图片文件 |
| F0.1.5 | 模型信息 | P1 | 显示模型基本信息（类型、类别数等） |
| F0.1.6 | 置信度调节 | P1 | 可调节置信度阈值滑块 |
| F0.1.7 | 摄像头推理 | P2 | 支持本地摄像头实时推理 |

- **模型加载**：`ultralytics.YOLO(model_path)`

### 验收标准

- [ ] 可以从文件系统选择并加载 .pt 模型
- [ ] 可以上传/选择图片进行推理

- [ ] 置信度阈值可调节且实时生效

## 非功能需求

| ID | 需求 | 说明 |
|----|------|------|
| NF1 | 启动时间 | 冷启动 < 10 秒 |
| NF2 | 内存占用 | 空闲 < 500MB |
| NF3 | 存储效率 | SQLite 数据库 < 100MB（不含模型和数据集） |
| NF4 | 跨平台 | Windows 10/11 为主要目标平台 |
| NF5 | Python 版本 | 支持 Python 3.10-3.12 |
| NF6 | 离线运行 | 首次 pip install 后完全离线可用 |

- 自动标注（VLM/SAM）
